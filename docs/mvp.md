# MVP 需求梳理（基于 PRD：效率管理四象限工具）

## 1. 产品目标（MVP）

在移动端用“四象限”帮助用户**快速分清优先级并落地执行**：一眼看到重要/紧急组合，并能快速创建、调整与完成任务。

## 2. MVP 范围（做什么）

### 2.1 核心页面

- **四象限主视图（Home）**
  - 屏幕 2x2 四象限布局
  - 每个象限展示：象限名称、任务数量、任务列表（可滚动）
  - 每个象限提供“新增任务”入口（+）

### 2.2 任务管理（CRUD）

- **新增任务**
  - 从某个象限点击“+”进入创建
  - 创建后默认归入该象限
- **编辑任务**
  - 点击任务进入编辑
  - 可修改全部字段（见 2.3）
- **删除任务**
  - 任务菜单删除（提供二次确认）

### 2.3 任务字段（数据模型）

- **必填**
  - 标题（title）
  - 象限（quadrant）
- **可选**
  - 描述（description）
  - 预计耗时（estimatedMinutes）
  - 截止日期（dueDate）
  - 标签（tags，逗号分隔）
- **状态**
  - 未开始 / 进行中 / 已完成 / 已延期
- **排序与置顶**
  - 同一象限内支持排序调整（MVP：上移/下移）
  - 支持置顶（Pinned 任务总排在该象限最前）

### 2.4 象限迁移（MVP 交付方案）

- PRD 原始期望：长按拖拽跨象限移动
- **MVP 1.0 交付**：任务菜单提供“移动到…”选项，选择目标象限完成迁移

## 3. MVP 非范围（暂不做）

- 智能建议（自动象限建议、迁移提醒、琐事识别）
- 今日聚焦视图与“下一步行动”推荐
- 番茄钟
- 数据统计与趋势报告
- 通知与提醒系统
- 语音输入、摇一摇、小组件、云同步、导出

## 4. 关键交互与体验要求（MVP）

- **效率优先**：新增与编辑流程尽量少步骤（建议使用底部弹窗/对话框表单）
- **响应速度**：常规任务操作（增删改、置顶、移动、上移/下移）应在 0.5s 内反馈
- **容量要求**：单用户 1000+ 任务仍可流畅浏览（列表虚拟化 + 数据库分页/合理查询）

## 5. Android 1.0 验收标准（建议）

- 四象限布局正确，颜色区分清晰
- 每象限任务数量准确
- 任务新增/编辑/删除可用，数据重启后仍存在
- 任务可在四象限间迁移
- 置顶与上移/下移排序可用，排序结果可持久化

## 6. 版本更新（后续迭代：1.0+）

> 说明：若不上架 Google Play，可采用“自建更新服务 + 下载 APK + 引导安装”的方式实现版本更新。

### 6.1 更新接口（update.json）

建议在你的服务器提供一个 JSON 文件（例如：`update.json`），返回当前最新版本信息：

```json
{
  "versionCode": 2,
  "versionName": "1.0.1",
  "apkUrl": "https://your-domain.com/xltool/app-release.apk",
  "notes": "修复若干问题，提升稳定性",
  "minSupportedCode": 1
}
```

字段说明：
- **versionCode**：必须递增，用于判断是否有更新
- **versionName**：展示用版本号
- **apkUrl**：可直接下载的 APK 链接
- **notes**：更新说明（可选）
- **minSupportedCode**：最低可继续使用的版本（低于此值可提示强制更新，后续可做）
